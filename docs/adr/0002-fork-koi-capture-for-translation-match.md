# Fork koi capture orchestration for translation match

The translation match exercise reuses the koi movement simulation and the bubble visual components (`KoiWordBubble`, `BubbleInstance`, `useBubbleAnimation`) as-is, but forks the capture orchestration hooks (`useKoiCaptureFlow`, `useKoiEscapeFlow`, `useKoiCaptureSharedState`, `useKoiRuntimeBridge`) into `wordLearning/translationMatch/koiCapture/` rather than parameterizing the table-exercise originals or extracting a shared capture core now. The two capture consumers diverge on resting position (screen center vs koi-zone center), match direction (English koi word → Spanish jellyfish label vs Spanish=Spanish), z-order (bubble above jellyfish vs below), swim zone (full screen vs koi rect), and elimination bookkeeping.

**Considered options:** parameterize the existing hooks with mode flags (leads to concentrated branching, the failure mode ADR-0001 rejected for the word-transformation core); extract a shared capture core now (premature — only two consumers, abstraction designed against one future consumer usually misses). Fork now, let the two diverge, and extract a shared core when a third capture consumer appears and the common shape is clear — the same sequence ADR-0001 followed (extract after the pattern proved out, not before).

> **Name update (ADR-0003):** The role names recorded above reflect the undersea theme as of the original decision. After the theme-agnostic reorganisation (ADR-0003), the generic roles are: **Roamer** (was koi — the roaming capturable-creature role) and **WordSprite** (was jellyfish — the floating word-display role). The capture hooks are now `useRoamerCaptureFlow`, `useRoamerEscapeFlow`, etc. The recorded decision is unchanged; only the role names have been generalised.
